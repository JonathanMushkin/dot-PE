Mission setup and goal:

i have a package at called dot-pe that is used to do parameter estimation for Gravitational Waves transient events. it is called dot-pe and is saved here:

/Users/jonatahm/Work/GW/dot-pe-future

the core of the calculation is a large number of matrix multiplications, using arrays that is loaded from files. this can trivially parallelized. Sadly, I/O bandwidth hinders making it faster using the several cores of a single machine. My goal is to split the time consuming matrix multiplications across several cores on a strong HPC server either using multi-processing / multi-threading, or even more brutally by submitting a few (~20?) short-term jobs. Namely, the server i have accessible is a LSF server.

the goal of this is to make a working proof-of-concept that can finish the same job using a fraction of the time.

algorithm:
the idea that i have is to slit the infernece.run() code into a linear part (setup), than replace the "for loop" that happens in the incoherent likelihood evaluation with some "swarm setup" to do many small LSF jobs / threads. maybe some commonly-accessed data must be saved to a temp folder for the purpose of making it work. than the results are combined and passed to the next "serial" stage, which is drawing extrinsic samples. 
When the extrinsic samples exist, the intrinsic samples and extrinsic samples can be combined coherently through as series of mostly matrix multiplications, again done by a "swarm", with needed shared data and potential re-combinations done at the end. 
then there is some standardization stage that could might as well be serieal.
